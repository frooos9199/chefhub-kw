# سيناريو عرض احترافي (Demo Flow) — عربي

هذا السيناريو مصمم للعرض أمام الشركة خلال 7–12 دقيقة.

## 0) قبل العرض (60 ثانية)
- افتح الموقع على جهاز العرض.
- جهّز 3 تبويبات (اختياري):
  1) صفحة الرئيسية
  2) صفحة /chef (قائمة الشيفات)
  3) لوحة الأدمن أو لوحة الشيف (حسب الدور)

## 1) مقدمة سريعة (30–45 ثانية)
- ChefHub منصة تربط العملاء مع الشيفات المحليين.
- الهدف: تسهيل التصفح والطلب + تمكين الشيف من إدارة أطباقه وطلباته.

## 2) تجربة العميل (3–4 دقائق)
1. افتح الصفحة الرئيسية:
   - أبرز الأقسام: الشيفات المميزين + أحدث الأطباق.
2. ادخل إلى قائمة الشيفات:
   - استخدم البحث/التصفية لإظهار سهولة الوصول.
3. افتح صفحة شيف محدد:
   - اعرض الأطباق، الصور، السعر، والتقييم.
4. أضف منتج إلى السلة وأكمل Checkout:
   - اعرض تأكيد الطلب و/أو صفحة نجاح الطلب.

## 3) تجربة الشيف (3–4 دقائق)
1. تسجيل الدخول كـ Chef.
2. لوحة الشيف:
   - عرض الطلبات الحالية.
3. إدارة الأطباق:
   - إضافة طبق جديد (صور + وصف عربي/إنجليزي + سعر).
   - تغيير التوفر (isAvailable) أو إخفاء الطبق (isActive) لعرض التحكم.
4. الإعدادات:
   - تحديث بيانات التوصيل/المحافظات.
   - تفضيلات الإشعارات.

## 4) تجربة الأدمن (2–3 دقائق)
1. تسجيل الدخول كـ Admin.
2. إدارة الشيفات:
   - الموافقة على شيف جديد أو إيقاف شيف.
   - شرح أن هذا يحكم ظهور الشيف للعامة.
3. البانرات/المحتوى:
   - تشغيل/إيقاف Banner (إن كان مفعل).
4. التقارير:
   - عرض صفحة التقارير/الإحصائيات (حسب الموجود).

## 5) فقرة “الأمان والجاهزية” (45–60 ثانية)
- صلاحيات واضحة حسب الدور.
- استقرار الاستعلامات (Indexes) على Firestore.
- إرسال إشعارات (In-app + Email/WhatsApp عند تفعيل المفاتيح).

## 6) أسئلة شائعة (Quick Q&A)
- هل يمكن إضافة أكثر من مطبخ/فروع؟
- هل هناك عمولة وإعدادات مالية؟
- كيف يتم اعتماد الشيفات؟
- هل الإشعارات تعمل فعلياً؟ (نعم عند تفعيل مفاتيح SendGrid/Twilio).
