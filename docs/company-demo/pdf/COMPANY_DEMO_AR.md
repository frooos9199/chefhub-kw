# ChefHub — ملف عرض الشركة

## نظرة عامة
ChefHub منصة تربط العملاء بالشيفات المحليين، وتتيح:
- للعميل: تصفح الشيفات/الأطباق والطلب بسهولة.
- للشيف: إدارة الأطباق والطلبات والإعدادات.
- للأدمن: اعتماد الشيفات وإدارة المحتوى والطلبات.

---

# الأدوار والصلاحيات

## 1) العميل (Customer)
**المزايا الأساسية**
- تصفح الشيفات والأطباق
- البحث والتصفية
- إضافة للسلة وإكمال الطلب
- متابعة حالة الطلب

## 2) الشيف (Chef)
**ما الذي يستطيع الشيف فعله؟**
- لوحة الشيف: متابعة وإدارة الطلبات
- إدارة الأطباق:
  - إضافة/تعديل طبق
  - التحكم في التوفر `isAvailable`
  - تفعيل/إخفاء الطبق `isActive`
  - رفع الصور (بحسب تفعيل التخزين)
- إعدادات الشيف:
  - تحديث بيانات التوصيل/الرسوم/المحافظات
  - تفضيلات الإشعارات

**ملاحظة للعرض:** قد لا يظهر الشيف للعامة حتى يتم اعتماده من الأدمن عبر `status`.

## 3) الأدمن (Admin)
**ما الذي يستطيع الأدمن فعله؟**
- لوحة الإدارة (Dashboard)
- إدارة الشيفات:
  - مراجعة طلبات التسجيل
  - اعتماد الشيف (Approved) أو رفضه أو إيقافه
  - التحكم في ظهور الشيف
- إدارة الطلبات (Monitoring)
- إدارة البانرات/المحتوى (إن كان مفعل)

# سيناريو عرض احترافي (7–12 دقيقة)

## 1) تجربة العميل (3–4 دقائق)
1. افتح الصفحة الرئيسية وأبرز أقسام الشيفات + الأطباق.
2. افتح قائمة الشيفات `/chef` واستخدم البحث/التصفية.
3. افتح صفحة شيف محدد `/chef/[id]` واعرض الأطباق.
4. أضف منتج للسلة وأكمل Checkout ثم نجاح الطلب.

## 2) تجربة الشيف (3–4 دقائق)
1. تسجيل الدخول كـ Chef.
2. لوحة الشيف: عرض الطلبات وتحديث حالتها.
3. الأطباق: إضافة طبق جديد أو تعديل طبق.
4. الإعدادات: مناطق التوصيل والرسوم وتفضيلات الإشعارات.

## 3) تجربة الأدمن (2–3 دقائق)
1. تسجيل الدخول كـ Admin.
2. اعتماد شيف جديد أو إيقاف شيف.
3. إدارة المحتوى/البانرات + التقارير (حسب الموجود).

# Checklist قبل العرض
- تشغيل: `npm install` ثم `npm run build` ثم `npm start`
- وجود حسابات ديمو (عميل/شيف/أدمن)
- وجود شيف حالته `approved` (أو `active` للبيانات القديمة)
- وجود أطباق: `isActive=true` و `isAvailable=true`

---

# Troubleshooting سريع
- الشيف لا يظهر:
  - تأكد من `status` و `isActive`.
- الأطباق لا تظهر:
  - تأكد من `isActive/isAvailable`.
  - راجع Firestore Indexes إذا ظهر “requires an index”.
- الإيميل/الواتساب:
  - يحتاج مفاتيح SendGrid/Twilio؛ بدونها النظام Debug ولن يرسل فعليًا.
